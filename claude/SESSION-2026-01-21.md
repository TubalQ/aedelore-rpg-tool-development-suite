### Rules Corrections Made
- **Attack Pool:** Changed from "Ability + Skill + Weapon bonus" to "Core Ability + Weapon Attack Bonus" (no skill in attack rolls)
- **Block Mechanic:** Fixed backwards logic. Now: Block value = damage absorbed (higher = better)

### Key Rule Clarifications
- **Weapons determine ability:** Strength (most melee), Dexterity (daggers, bows), Str/Dex choice (Katana)
- **Defense is reactive:** Player chooses Dodge/Parry/Block/Take Hit when attacked
- **Block absorbs damage:** Success = absorb up to Block value, rest goes through, shield takes blocked damage to HP
- **No "melee skill":** Attack pools don't include skills, only Core Ability + Weapon Attack Bonus

## Session 2026-01-19: DM Session Tool Implementation

### What was built
A complete DM (Dungeon Master) Session Tool for managing RPG campaigns and sessions.

### Files Created
- `/opt/aedelore/html/dm-session.html` - DM tool web page with 10 tabs
- `/opt/aedelore/html/js/dm-session.js` - JavaScript for all DM tool functionality

### Files Modified
- `/opt/aedelore/api/db.js` - Added `campaigns` and `sessions` tables
- `/opt/aedelore/api/server.js` - Added 11 new API endpoints for campaigns/sessions
- `/opt/aedelore/html/character-sheet.html` - Added "DM Tools" button (line 108-113)
- `/opt/aedelore/html/js/main.js` - Modified `updateAuthUI()` to show/hide DM button

### Database Schema Added
```sql
campaigns (id, user_id, name, description, created_at, updated_at)
sessions (id, campaign_id, user_id, session_number, date, location, status, data, created_at, updated_at)
```

### API Endpoints Added
**Campaigns:**
- GET /api/campaigns
- GET /api/campaigns/:id
- POST /api/campaigns
- PUT /api/campaigns/:id
- DELETE /api/campaigns/:id

**Sessions:**
- GET /api/campaigns/:campaignId/sessions
- GET /api/sessions/:id
- POST /api/campaigns/:campaignId/sessions
- PUT /api/sessions/:id
- PUT /api/sessions/:id/lock
- PUT /api/sessions/:id/unlock
- DELETE /api/sessions/:id

### Features
- Multiple campaigns per user
- Multiple sessions per campaign
- Session locking (active/locked states)
- 10 tabs: Info, Players, Scenes, Decisions, Items/Clues, NPCs, Combat, Read-Aloud, Notes, Loot
- Quick "Add" buttons for all dynamic content
- Auto-save every 30 seconds
- JSON export
- Print function

### Access
- Login required to see "DM Sessions" button in character-sheet.html
- Button appears in header after "App" section when logged in

### Testing
After API restart, user can:
1. Go to character-sheet.html
2. Login
3. Click "DM Sessions" button
4. Create campaigns and sessions
